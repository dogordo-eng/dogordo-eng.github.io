Metodologia: eXtreme Go Horse (XGH)

Orgulhosamente adotamos o XGH como metodologia oficial. Move fast, break things, frita tudo.

1. Pensou, não é XGH

Nem pensa no recheio, já joga na massa e frita. Se parar pra pensar, o pastel esfria e o deploy atrasa.

2. Três formas de resolver um problema

O jeito certo, o errado, e o XGH — que é igual o errado, mas o pastel sai mais rápido e o cliente nem percebe.

3. Testes são para os fracos

Se compilou e se fritou, tá pronto. Provar o pastel antes de servir é opcional. O cliente testa em produção.

4. Código funcionando: não altere

Pastel que tá vendendo não muda a receita. Se o sistema roda e o pastel vende, mexer nisso é heresia.

5. XGH é totalmente reativo

Pastel só queima quando alguém percebe. Bug só é bug quando o cliente reclama. Até lá, é feature.

6. Não existe refatoração em XGH

Pastel que saiu torto não volta pra massa. Vira promoção do dia e segue o jogo.
